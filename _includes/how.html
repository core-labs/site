How

An old operating system's concept in a new light.

We're implementing a distributed shared memory infrastructure called gallocy. At its core, gallocy is user-space C/C++ software library that implements a network-aware virtual memory management system. Just as an operating system manages the memory of a physical machine, gallocy manages the memory of a distributed virtual machine.

The gallocy framework joins your data center into one logical computer. What was a warehouse full of servers is now a warehouse full of resources that can be thought of and treated just like CPU and RAM of a single server.
Combined with a gallocy-aware implementation of the POSIX threads interface and a novel memory coherence protocol, we're creating a distributed computer that can address a conceptually infinite amount of memory and processing cores, while preserving the process abstraction. Run a single process across all of your machines.

The possibilities are endless. Build distributed in-memory filesystems, key-value stores, graph databases, relational query engines, and more, by leveraging the gallocy framework.
